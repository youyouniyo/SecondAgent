API 장애 대응 절차

이 문서는 운영 중 발생할 수 있는 API 장애에 대해 담당자가 적절히 확인하고 조치한다 는 내용을 설명합니다.

1. 장애 확인

알림을 수신하면 모니터링 대시보드에서 상태를 확인합니다. 문제가 있으면 관련 팀에 공유합니다.

  • Grafana 대시보드 확인
  • 최근 배포 이력 확인
  • 영향 범위 확인

2. 복구 진행

장애 원인이 최근 배포와 관련되어 있으면 롤백을 검토합니다. 필요 시 적당한 방법으로 복구한다.
